# 贡献指南

感谢你对本项目的兴趣！我们欢迎任何形式的贡献，包括但不限于报告问题、提交功能请求、完善文档和提交代码。

## 问题反馈 (Issue)

提交问题时，请按照以下模板填写：

### 1. 遇到了什么问题？
请详细描述你遇到的问题，包括：
- 期望行为
- 实际行为
- 错误信息（如有）

### 2. 预期工作的形式？
描述你希望该功能或修复如何工作。

### 3. 解决该问题的思路（若有）
如果你已经思考过解决方案，请分享你的想法。即使不完整，也能帮助我们更好地理解问题。

---

## 提交拉取请求 (Pull Request)

提交 PR 时，请按照以下模板填写：

### 1. 遇到了什么问题？
描述你修复的问题或添加的功能背景。

### 2. 预期工作的形式？
描述修改后的预期行为。

### 3. 解决该问题的方案
详细说明你的修改内容和实现思路。

### 4. 是否与当前上游冲突？
- [ ] 已检查与上游仓库的兼容性
- [ ] 无冲突或已解决冲突

---

## 开发环境设置

```bash
# 克隆仓库
git clone https://github.com/afoim/fuwari.git
cd fuwari

# 安装依赖
pnpm install

# 启动开发服务器
pnpm dev
```

## 代码规范

- 使用 Biome 进行代码格式化：`pnpm format`
- 使用 Biome 进行代码检查：`pnpm lint`

## 提交信息规范

请使用清晰描述性的提交信息：
- `feat: 添加新功能`
- `fix: 修复问题`
- `docs: 更新文档`
- `style: 代码格式调整`
- `refactor: 代码重构`
